# 知乎吹牛逼

面向 2026 知乎「AI 脑洞实验室」黑客松灵感引擎赛道的前端 Demo。

它不是一个普通测试，也不是 AI 写作器。用户从一个真实想问的知乎式问题进入，连续做 3-5 个选择，系统根据选择判断用户更偏向哪一种答案，再给出：

- 一条知乎相似高赞问题链接
- 一句高赞回答式浓缩回声
- 一句适合保存和分享的结果判词
- 一张自动生成的小卡片

## 产品概念

「知乎吹牛逼」里的“吹”，不是判断用户是不是在吹牛逼，而是借知乎 15 年真实问答里的回声，给用户心里已经偏向的答案递一口气。

用户不是来听大道理的，他只是想知道：有没有一群人、高赞回答、过来人，也站在自己这一边。

## 当前玩法

首页是一张漂浮问题星图，包含 20+ 个常见知乎式问题：

- 要不要裸辞？
- 该不该复合？
- 我要不要分手？
- 要不要转行？
- 要不要创业？
- 我炒股能不能赚钱？
- 为什么努力没有结果？
- 我是不是太普通了？
- 要不要离开这座城市？
- 为什么我总在讨好别人？

每条精线路包含 3 道选择题，背后判断用户此刻更需要鼓励、降温、认同、止损、行动还是体面收尾。结果页会给出知乎搜索链接和一句可保存的小卡片。

## 本地运行

```bash
python3 -m http.server 5173
```

打开 `http://localhost:5173`。

## 本地自检

```bash
bash scripts/smoke-test.sh
```

测试会检查：

- JS 语法
- 静态构建
- 首页加载
- 路由参数
- 自定义问题匹配
- 浏览器真实点击问题、选项、结果页、复制、小卡片
- 移动端水平溢出

## API 与 OAuth

当前版本优先保证体验完整：

- 前端结果页会给出知乎相似问题搜索链接。
- `api/challenges.js` 保留为知乎 API 代理适配层，可继续接官方热榜/问题/高赞回答接口。
- “用知乎登录”当前为授权演示状态；拿到正式 OAuth 回调配置后，可把用户授权身份、兴趣和知乎 API 数据接入同一套分叉结果。

## 部署

这是静态站点，可直接部署到 GitHub Pages。仓库内已包含 `.github/workflows/pages.yml`。

生成单文件部署包和提交压缩包：

```bash
bash scripts/package-submission.sh
```

## 演示入口

- `/`：首页问题星图
- `/?route=stock`：直接进入“我炒股能不能赚钱？”
- `/?ask=我想辞职开咖啡店`：自定义问题自动匹配线路
