# 知乎吹牛逼鉴定局持续推进计划

## 当前目标

把「知乎吹牛逼鉴定局」从本地静态 Demo 推进到可线上访问、可路演展示、可按知乎 Hackathon 要求提交的作品。

## 每小时自检节奏

已经设置每小时线程唤醒。每次唤醒检查：

- 本地代码是否可运行
- Git 提交和远程发布状态
- 官方文档/API 是否已经拿到关键内容
- Demo 是否比上一小时更接近可提交
- 下一小时最关键的一件事是什么

## 官方文档需要提取的信息

来自用户提供的飞书链接：

- 参赛流程及 API 调用
- 活动规则文档
- 建队登记
- 作品提报说明

重点要确认：

1. API base URL、鉴权方式、token 获取方式
2. 可调用的数据范围：热榜、问题、回答、搜索、推荐、社区内容特征
3. 请求频率和额度限制
4. 是否允许把 API 返回内容展示在公开 Demo 中
5. 作品提交入口、提交材料、最终截止时间
6. 是否要求代码仓库、演示视频、PPT 或说明文档
7. 是否允许线上部署到 GitHub Pages / Vercel / Netlify

## 产品升级路线

### V1 已完成

- 纯静态游戏页面
- 5 局真假高赞鉴定
- 三类判断：社区高赞风格、AI 仿深刻、纯吹牛逼
- 每局解释为什么像高赞/AI/纯吹
- 结果人格卡
- Canvas 分享图
- 临场造题模块

### V2 下一步

- 增加「知乎热榜实验室」入口
- 支持从官方 API 或本地 fallback 题库读取问题
- 结果页增加「我被哪种话术骗了」雷达图
- 增加多人挑战链接参数，让同一套题可分享
- 改进移动端分享卡，适合微信截图传播

当前已完成 V2 的三个基础能力：热榜实验室、本地 fallback 题库、分享挑战链接。

结果页已补上创作者灵感转化能力，能把用户的易感话术转成可写角度、高赞开头和评论区钩子。

### V3 路演增强

- 做评委演示模式：固定题目、固定节奏、可一键跳结果页
- 增加创作者模式：把高赞结构拆成标题、开头、反方观点、评论区钩子
- 增加 API 状态面板：展示实时热榜/题库来源
- 准备 90 秒 Demo 讲稿和 3 分钟路演讲稿

## 发布计划

优先路径：

1. 创建 GitHub 仓库 `NewRudy/zhihu-chuiniubi-judge`
2. 推送本地代码
3. 用 GitHub Pages 部署静态版
4. 如果知乎 API 需要隐藏 token，则改用 Vercel/Netlify Serverless 代理
5. 最终提交在线链接、仓库链接、说明文档和演示材料

已增加 `scripts/build-dist.py`，可随时把当前源码打包成 `dist/index.html` 单文件版本，降低静态部署摩擦。

## 当前阻塞

- 本机没有 `gh` 命令
- Git 没有可用的 GitHub 凭据
- 当前 GitHub 连接器能访问账号和已有仓库，但没有创建新仓库工具
- 飞书文档需要登录态或正文内容才能解析 API 细节
