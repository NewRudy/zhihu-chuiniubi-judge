# 知乎吹牛逼：黑客松提交说明

## 一句话介绍

从一个真实想问的知乎式问题出发，做几道选择题，找到一条和你站在同一边的高赞回声。

## 产品定位

「知乎吹牛逼」是一个认同感分叉游戏。它不直接替用户做决定，而是把常见知乎问题做成轻互动：用户连续点击几个选项后，系统判断他心里更偏向哪种答案，再给出知乎相似问题、高赞回答式摘要和一张可保存的小卡片。

## 为什么适合灵感引擎赛道

知乎 15 年社区数据里沉淀了大量真实困惑、人生选择、后悔复盘和高赞回答。这个 Demo 把这些内容从“搜索答案”变成“交互式认同感”：用户不是被动看回答，而是在选择中认出自己想相信的那一派。

它既有普通用户愿意分享的娱乐性，也能扩展成创作者工具：把真实困惑、高赞回声和用户情绪分叉转成选题、标题和回答角度。

## 核心玩法

1. 首页展示 20+ 个漂浮知乎式问题。
2. 用户点一个问题，或自己输入一个问题。
3. 系统匹配最相近线路，进入 3 道选择题。
4. 根据用户选择判断他更偏向鼓励、降温、认同、止损、行动或收尾。
5. 结果页展示知乎相似问题链接、高赞回声、最后一句判词。
6. 用户可以复制结果句，或保存分享卡片。

## 已精写线路

- 要不要裸辞？
- 该不该复合？
- 我要不要分手？
- 要不要转行？
- 要不要创业？
- 我炒股能不能赚钱？
- 为什么努力没有结果？
- 我是不是太普通了？
- 要不要离开这座城市？
- 为什么我总在讨好别人？

其他漂浮问题会映射到相近线路，保证首屏看起来丰富，同时让核心体验足够精。

## 传播机制

- 问题真实：每个入口都像用户可能匿名发到知乎的问题。
- 选择有代入：选项像用户心里的辩解，而不是普通问卷。
- 结果有认同：最后给出的不是标准答案，而是用户想找到的那一派高赞回声。
- 卡片可分享：一句短判词 + 问题 + 品牌名，适合截图传播。

## API 接入设计

当前静态版已稳定可演示，结果页给出知乎相似搜索链接。正式接入后：

- OAuth 登录获取用户授权身份和兴趣信号。
- 根据用户选择和问题线路，匹配知乎真实问题。
- 拉取高赞回答链接、摘要和统计数据。
- AI 只负责把高赞回答压缩成 1-2 句“高赞回声”。
- 接口失败时保留本地精写 fallback，保证路演不中断。
